Pattern Matching (JS)

A pattern matching DSL embedded in plain JavaScript.

How it works

The core idea is simple: match takes a value followed by alternating pattern/handler pairs. It tries each pattern top to bottom, and when one fits, it calls the corresponding handler with whatever the pattern captured. This all happens in plain JavaScript--no syntax extensions, no build step. The DSL is just functions.

match(42,
  0,    () => "zero",
  _,    x  => "got " + x
)

Literals match by equality. The wildcard _ matches anything and passes the value to the handler as a parameter. If no pattern matches, it throws a MatchFailure.

Arrays

Array patterns match element by element. If you pass an array as a pattern, the value must also be an array, and each element is matched against the corresponding pattern in the array. This gives you destructuring for free.

match([1, 2, 3, 4],
  [1, many(_), 4],  middle => middle
)

many(pattern) matches zero or more consecutive elements that fit the inner pattern, collecting them into an array. rest() captures everything remaining in the list from that point on. Between the two, you can express most list-processing patterns without writing recursive code.

Class destructuring

This is where it gets interesting. If you define a deconstruct() method on a class that returns an array of its components, you can use inst(ClassName, ...patterns) to match and pull apart instances. Each pattern position corresponds to one element of the returned array.

class Point {
  constructor(x, y) { this.x = x; this.y = y }
  deconstruct() { return [this.x, this.y] }
}

match(new Point(3, 4),
  inst(Point, 0, _),  y => "on y-axis at " + y,
  inst(Point, _, _),   (x, y) => "(" + x + ", " + y + ")"
)

The first arm checks if it's a Point whose x-coordinate is 0. The second matches any Point and binds both coordinates. Because deconstruct is just a method you write, you control exactly what gets exposed to the pattern matcher. The interpreter in the functional language playground actually uses this same mechanism internally--it pattern-matches on AST nodes using inst and deconstruct.

Guards and ranges

A function used as a pattern acts as a guard: it receives the value and returns true or false. You can also combine a structural pattern with a boolean check using when:

match(7,
  when(_, x => x > 10),  x => "big",
  _,                       x => "small"
)

range(min, max) matches any number in the inclusive range. These are convenience tools--you could do the same thing with a function pattern, but they read more clearly.
