Functional Language (F)

F is an ML-like language with pattern matching, data constructors, and list comprehensions.

Let bindings

Everything in F is an expression that produces a value. let binds a name to a value for the duration of the body after in. The binding is scoped--once the body finishes, the name goes away. This means you can nest let expressions freely without worrying about names leaking out.

let x = 5 in x + 1

When you need recursion, use let rec. This lets the bound name refer to itself in its own definition. Without rec, the name isn't in scope yet when the right-hand side is being evaluated, so a plain let can't define recursive functions.

let rec fac = fun n -> if n = 0 then 1 else n * fac (n - 1) in
  fac 5

Functions

Functions are written with fun and an arrow. When you write multiple parameters, each one creates a nested function--fun x y -> x + y is really fun x -> fun y -> x + y. This currying means you can partially apply: pass one argument and get back a function that's waiting for the rest.

Application is by juxtaposition. You write f 5, not f(5). This keeps things compact, especially when chaining: f 5 + 17 applies f to 5, then adds 17 to the result.

Functions close over their environment. When you define a function inside a let, it captures the bindings that were in scope at the time. The resulting closure carries those bindings with it, so it works correctly even when called from a different scope.

Data constructors and pattern matching

Constructors start with an uppercase letter and wrap values into tagged data. Some 5 creates a datum tagged Some containing 5. None is a constructor with no payload. You can nest them: Cons (1, Cons (2, Nil)) builds a linked list. The bracket syntax [1; 2; 3] is sugar for exactly that chain of Cons and Nil.

match lets you branch on the shape of a value. Each arm is a pattern followed by an arrow and a body. Patterns can be literals, wildcards (_), variables (which bind the matched value), constructors, or tuples. The interpreter tries each arm top to bottom and takes the first one that fits.

match Cons (1, Nil) with
| Cons (x, _) -> x
| Nil -> 0

Patterns also work in let bindings and function parameters. let (a, b) = (1, 2) in a + b destructures a tuple directly. fun (x, y) -> x + y takes a tuple argument and pulls it apart.

Tuples and lists

Comma-separated expressions in parentheses make a tuple: (1, true, 3). Tuples can hold any mix of types and can be destructured anywhere patterns are allowed.

Lists use semicolons in brackets: [1; 2; 3]. You can also build them from comprehensions. A list comprehension maps an expression over a list, with an optional filter:

[x * x | x <- [1; 2; 3; 4; 5], x > 2]

This iterates over the list, binds each element to x, keeps only those where x > 2, and collects x * x for each one. The result is [9; 16; 25].

Delay and force

delay wraps an expression without evaluating it, creating a thunk. The expression just sits there until someone calls force on it, at which point it evaluates and caches the result. Calling force again returns the cached value without re-evaluating. This gives you lazy evaluation in a language that's otherwise eager--useful for things like infinite data structures or expensive computations you might not need.
