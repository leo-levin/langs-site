Logic Language (L)

L is a Prolog-like language with unification and backtracking search.

Facts and rules

A program in L is a collection of facts and rules, followed by a query. You're not writing a sequence of instructions--you're stating what's true, and then asking a question about it.

A fact is a clause ending in a period. It says something is unconditionally true. parent(tom, bob). means Tom is Bob's parent, full stop. A rule adds conditions: the head is true if all the goals in the body are true. The body comes after :- and goals are separated by commas.

parent(tom, bob).
parent(bob, ann).
grandparent(X, Z) :- parent(X, Y), parent(Y, Z).

Variables and queries

Variables start with an uppercase letter. Symbols--the concrete atoms of your program, like tom or parent--start with lowercase. This distinction matters: variables are placeholders that the interpreter fills in, while symbols are fixed values.

A query ends with ? instead of a period. When you ask grandparent(tom, X)?, the interpreter searches for all values of X that make the query true. It does this by unification--matching the query against rules, binding variables to terms, and backtracking when a path doesn't work out. If there are multiple solutions, you get all of them.

Unification

Unification is the core mechanism. When the interpreter tries to match a goal against a rule's head, it finds a substitution--a mapping from variables to terms--that makes them identical. If parent(X, ann) is matched against parent(bob, ann), the substitution is {X = bob}. If two terms can't be made to match--different symbols, different arities--unification fails and the interpreter backtracks to try another rule.

Lists

Lists are written with brackets: [1, 2, 3]. Under the hood, they desugar to nested cons/nil structures, but you don't need to think about that. The | separator splits a list into its head and tail, which is how you recurse over lists in patterns:

member(X, [X | _]).
member(X, [_ | T]) :- member(X, T).
member(2, [1, 2, 3])?

The first rule says: X is a member of a list if it's the head. The second says: X is a member if it's a member of the tail. The underscore _ is a wildcard that matches anything without binding it.

Arithmetic

L doesn't evaluate arithmetic automatically--plus(2, 3) is just a term, not 5. To actually compute, use is. It evaluates the right-hand side as an arithmetic expression and unifies the result with the left-hand side. The operations are plus, minus, times, div, and mod.

double(X, Y) :- Y is times(X, 2).
double(5, Y)?

Cut, assert, retract

cut prunes the search tree. When the interpreter reaches a cut, it commits to the current choice and won't backtrack past it. This is useful for writing "if-then-else" style logic or preventing redundant search.

assert and retract modify the program at runtime. assert(likes(bob, pizza)) adds a new fact. retract(likes(bob, pizza)) removes the first matching one. This lets programs build up and modify their own knowledge base during execution.
